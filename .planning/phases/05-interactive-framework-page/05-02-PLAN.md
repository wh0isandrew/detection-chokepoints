  <action>
Create `assets/js/framework-graph.js` as an IIFE (immediately invoked function expression) wrapping all logic. The script reads `window.__GRAPH_DATA` and renders a D3 v7 force-directed graph.

**Data contract:**
- `window.__GRAPH_DATA.nodes` — array of `{ id, type, name, group, vars?, maturity?, url? }`
- `window.__GRAPH_DATA.links` — array of `{ source, target }` where source/target are node IDs
- `type` is one of: `"tactic"`, `"chokepoint"`, `"technique"`
- `group` is a slugified tactic name (e.g., `"credential-access"`, `"initial-access"`)

**Guard clause:** If `window.__GRAPH_DATA` is undefined or `#graph-svg` element doesn't exist, return early without error.

**Color reading:** Read CSS custom properties at runtime using `getComputedStyle(document.documentElement).getPropertyValue('--varname').trim()`. Read these once at init:
- `--critical` for chokepoint node color
- `--accent` for technique node color (NOTE: site's --accent is orange #f0883e, not blue)
- `--medium` for tactic node color
- `--border` for link stroke and arrow marker fill
- `--text-muted` for label text
- `--text-dim` for secondary text
- `--bg-card` for variation count badge background
- `--bg-surface` for tooltip background
- `--text` for tooltip title text

**Node sizing:**
- `tactic`: radius 22, fill-opacity 0.25, stroke-width 2
- `chokepoint`: radius 16, fill-opacity 0.85, stroke-width 1.5
- `technique`: radius 10, fill-opacity 0.85, stroke-width 1.5

**Force simulation:**
- `forceLink`: id accessor = `d.id`, distance = 140 for links involving tactics, 80 for others
- `forceManyBody`: strength = -400 for tactics, -250 for chokepoints, -120 for techniques
- `forceCenter`: center of SVG (width/2, height/2)
- `forceCollide`: radius = node radius + 8

**SVG setup:**
- Append SVG to `#graph-svg` element
- Width = container's clientWidth, height = 520
- Set viewBox = `[0, 0, width, height]`
- Define arrow marker in `<defs>` (id="arrow", viewBox="0 -3 6 6", markerWidth/Height=6)

**Links:** `<line>` elements, stroke = `--border` color, stroke-width 1.2, stroke-opacity 0.5

**Nodes:** `<g>` groups with drag behavior (d3.drag with alphaTarget 0.3 on start, null fx/fy on end)
- Each group contains a `<circle>` with radius/fill/opacity per type
- Each group contains a label `<text>` below the circle:
  - Tactic: full name, 11px, font-weight 600, dy = radius + 14
  - Technique: ID string (e.g., "T1555.003"), 9px, dy = radius + 12
  - Chokepoint: name truncated to 20 chars + "..." if longer, 9px, dy = radius + 12
- Chokepoint nodes with `vars > 0` get a variation count badge: small circle at (12, -12) radius 8 with count text

**Tooltip behavior (on mouseover):**
- Select `#graph-tooltip` element
- Set `.tt-title` to node name
- Set `.tt-meta` based on type:
  - chokepoint: `"{vars} variations · {maturity} maturity"`
  - technique: `"{id} · {name}"`
  - tactic: `"Tactic group"`
- Position tooltip at `event.offsetX + 15`, `event.offsetY - 10`
- Add class `visible` to tooltip
- **Highlight connections:** Build a Set of connected node IDs (the hovered node + all direct neighbors via links). Set fill-opacity to 0.95 for connected nodes, 0.15 for others. Set link stroke-opacity to 0.8 for connected links, 0.07 for others.

**Tooltip hide (on mouseout):**
- Remove `visible` class from tooltip
- Reset all node fill-opacities to defaults (0.25 for tactics, 0.85 for others)
- Reset all link stroke-opacities to 0.5

**Click behavior:**
- On chokepoint node click: if `d.url` exists, navigate with `window.location.href = d.url`

**Tactic filter buttons:**
- Select all `.graph-btn[data-filter]` elements
- On click: remove `active` class from all buttons, add to clicked button
- If filter is `"all"`: show all nodes and links, reset opacities
- Otherwise: build a Set of visible node IDs where `node.group === filter`, then expand the set by adding all nodes connected via links. Hide nodes/links not in the visible set (`display: none`). Show matching ones (`display: block`).
- Restart simulation with alpha 0.5

**Responsive:** On window resize, update SVG width to container clientWidth and restart simulation. Debounce resize handler by 250ms.

**Tick handler:** Update link x1/y1/x2/y2 and node transform on each simulation tick.

**Deduplication:** The Liquid template may produce duplicate link entries (e.g., same technique linked to same tactic via multiple chokepoints). Deduplicate links at init by building a Set of `source + "|" + target` strings and filtering.
