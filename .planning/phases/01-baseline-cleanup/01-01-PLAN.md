<task type="auto">
  <name>Task 3: Fix Liquid syntax error in emulation language assignment and verify clean strict build (BSLN-03)</name>
  <files>_layouts/chokepoint.html</files>

  <read_first>
    - _layouts/chokepoint.html — read lines 1550-1560 (the emulation block with the syntax error) before touching anything
  </read_first>

  <action>
Line 1557 of `_layouts/chokepoint.html` contains:

```liquid
{% assign emu_lang = "language-" | append: (emu.Language | downcase | default: "plaintext") %}
```

Parentheses wrapping a filter chain are invalid Liquid syntax. Jekyll emits a warning for every chokepoint that has an emulation block. The fix is a two-step assign — compute the language string first, then append:

Replace line 1557 with these two lines:

```liquid
          {% assign emu_lang_raw = emu.Language | downcase | default: "plaintext" %}
          {% assign emu_lang = "language-" | append: emu_lang_raw %}
```

The `emu_lang` variable is consumed on line 1558:
```liquid
<pre class="sigma-code rounded-lg p-4 overflow-x-auto text-[.8rem]"><code class="{{ emu_lang }}">
```
That line is unchanged.

After making the fix, run:
```
jekyll build --strict 2>&1
```

The build MUST exit zero with no warnings. If any warnings remain, read the output carefully and fix each one before considering this task done.
  </action>
